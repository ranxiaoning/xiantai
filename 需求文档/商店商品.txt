# 黑市商品 V1

黑市由“贪婪的渡劫者”经营，使用局内货币“灵石”。V1 目标是完成购买闭环与基础生效：卡牌、物品、宝物、删牌、升级都能交易；丹药在地图使用即时生效；阵法可激活并记录；符箓可购买、入包、展示，战斗使用后续接入。

## 一、商品刷新

| 区域 | 每次进货 | 规则 |
|---|---:|---|
| 卡牌 | 3 张 | 从 `CardDatabase` 抽取，按当前地图层数分早/中/后期稀有度池，不重复 |
| 物品 | 4 个 | 统一包含丹药、符箓、阵法，按 `min_floor` 过滤，不重复 |
| 宝物 | 2 个 | 不展示本局已拥有宝物，同一商店内不重复 |
| 服务 | 删除/升级 | 删除卡牌每次进入黑市最多 1 次；升级可多次，只能升级未升级卡 |

层数分段：

| 阶段 | 层数 | 卡牌稀有度池 | 商品解锁 |
|---|---|---|---|
| 早期 | 1-5 | 黄品为主，少量玄品 | `min_floor <= 5` |
| 中期 | 6-10 | 黄/玄为主，少量地品 | `min_floor <= 10` |
| 后期 | 11+ | 玄/地为主，少量天品 | `min_floor <= 当前层` |

## 二、价格

| 项目 | 黄品 | 玄品 | 地品 | 天品 |
|---|---:|---:|---:|---:|
| 购买卡牌 | 50 | 80 | 150 | 300 |
| 升级卡牌 | 30 | 50 | 80 | 150 |

删除卡牌费用按本局已使用次数递增：50、75、100、125……每次进入黑市最多删除 1 张；牌组只剩 1 张时不可删除。

## 五、实现接口

| 接口 | 说明 |
|---|---|
| `ShopDatabase.generate_stock(floor, owned_artifact_ids, seed)` | 返回 `{cards, items, artifacts}` |
| `ShopDatabase.get_card_price(card)` | 按卡牌稀有度返回购买价格 |
| `ShopDatabase.get_upgrade_price(card_id)` | 按卡牌稀有度返回升级价格 |
| `ShopDatabase.get_remove_price(remove_count)` | 返回删牌费用 |
| `GameState.spend_spirit_stones(cost)` | 灵石不足时不改变状态 |
| `GameState.buy_shop_card(card_id, price)` | 扣费并加入牌组 |
| `GameState.buy_shop_item(item, price)` | 背包未满时扣费并入包 |
| `GameState.buy_shop_artifact(artifact, price)` | 扣费并加入宝物栏 |
| `GameState.remove_deck_card_at(index, price)` | 扣费、删牌、递增删牌次数 |
| `GameState.upgrade_deck_card_at(index, price)` | 扣费并将卡牌 id 追加 `+` |
| `GameState.use_consumable(index, "map")` | 丹药生效并消耗；阵法激活不消耗；符箓提示暂未开放 |

## 六、测试与手动验证

自动化：

- `TestShopDatabase`：数量、分类、库存、价格、宝物去重。
- `TestShopTransactions`：扣费失败不变更、买卡/物品/宝物、背包上限、删牌、升级、地图使用丹药/阵法。
- `TestScriptIntegrity`：`ShopDatabase.gd`、`ShopScene.gd`、`Shop.tscn`、Autoload 注册。

手动验证：

- 从地图点击黑市节点进入 `scenes/Shop.tscn`。
- 分别购买卡牌、物品、宝物，确认灵石、牌组、背包、宝物栏同步。
- 删除卡牌每次进入黑市最多 1 次，费用随本局次数递增。
- 升级卡牌后牌组中对应 id 追加 `+`，已升级卡不能重复升级。
- 返回地图后，丹药点击生效并移除；阵法点击后高亮当前阵法且不移除；符箓点击只提示。
