# 卡牌批量生成器代码流程解析 (gen_all_cards.py)

本文档旨在详细解析 `assets\card\gen_all_cards.py` 脚本的执行流程和内部逻辑，以便于未来的功能扩展与手动代码调试。

## 1. 整体设计思路
脚本的核心思想是：**“数据 + 模板 + 素材 = 最终成品”**。
通过读取存储有卡牌所有属性数值的 `all_card.json` 文件，然后基于一张固定的背景图片 `template.png`，利用 `Pillow (PIL)` 图像处理库，将相应的原画（插图）和文字动态地绘制合成到指定坐标，最终输出所有卡牌的完整图鉴。

## 2. 核心执行流程

脚本从上到下的执行顺序如下：

### 阶段一：环境与资源初始化
1. **路径初始化：** 自动获取当前脚本所在路径（`BASE`），并推导出所需的模板文件、JSON配置、插画目录（`art/`）和输出目录（`generated/`）。代码会自动检测并创建不存在的目录。
2. **加载字体与常量设定：** 
   - 载入系统级的 `.ttf` 字体文件（黑体、楷体）。
   - 定义所有的核心常量，包括各文本区域的**坐标锚点**（X, Y）、**相对比例**、**字体大小**和**颜色字典**（根据类型或稀有度动态匹配颜色）。

### 阶段二：数据读取 (`main()` 函数入口)
1. **解析 JSON 数据：** 脚本通过 `open()` 加载外部的 `all_card.json` 文件，并将其转化为 Python 字典。
2. **遍历生成：** 提取字典中的 `cards` 列表，开始通过 `for` 循环遍历每一条卡牌数据。
3. **分发任务：** 将每一条具体的卡牌数据（字典形式）传入核心的渲染函数 `render_card(card)` 中。

### 阶段三：图像合成渲染 (`render_card(card)` 函数)
在渲染一张具体的卡牌时，程序按以下步骤层层叠加图像与文字：
1. **准备画布：** 读取底图 `template.png`，并转换成 `RGBA` 色彩模式以支持后续支持透明度的合成操作。
2. **加载字体对象：** 根据设定好的字体大小和样式路径，实时实例化不同的字体对象（名称用粗体，描述用楷体等）。
3. **合成原画：** 
   - 根据当前处理的卡牌 `id`（例如：`05`），去 `art/` 目录下寻找对应的 `05.png`。
   - 如果图片存在，将其读入、缩小并调整为大约 `1000x940` 大小。
   - 使用 `img.paste` 连同本身的 Alpha 通道（作为 mask）粘贴到卡牌的卡槽区域。
4. **渲染文本属性（利用辅助绘制函数）：**
   - **名称颜色判断：** 从 `RARITY_COLORS` 字典中获取对应稀有度（天品、地品等）的颜色。
   - **绘制消耗（`glowing`函数）：** 为左侧的"灵力消耗"和右侧的"道行消耗"绘制带有黄色渐变发光效果的数字。
   - **绘制标题与类型（`centered`函数）：** 绘制带有轻微阴影的卡牌名字（居中靠上）和法术类型（居中靠下）。
   - **绘制效果描述（`wrapped_centered`函数）：** 这是一个**自适应算法**函数，会根据行宽自动在标点符号处断句折行，并在划定的区域框内**水平且垂直居中对齐**，确保多行文本排版美观。
5. **保存输出：** 剔除 Alpha 通道（转换为 `RGB`），按照 `<id>_<name>.png` 的命名规则（如 `01_劈山.png`），保存进 `generated/` 文件夹。

## 3. 调试与排错指南

如果需要**手动调试**代码，请特别关注以下几点：
- **字体报错（OSError: cannot open resource）：** 如果在非 Windows 环境（如 Linux/Mac）执行，找不到 `C:/Windows/Fonts/` 路径时，请修改 `FONT_BOLD` 和 `FONT_KAITI` 到您系统内实际存在的中文字体路径。
- **坐标偏移错位：** 当前的 `LING_CX`, `DESC_TOP` 等坐标点完全绑定于一张 `1536×2752` 像素的原模板。若替换了不同分辨率的新模板图，所有的锚点必须重新校准。
- **原画糊掉或拉伸：** `art_img.resize((1000, 940))` 是强行拉伸指定尺寸，如果原画比例不是近乎正方形，建议修改为 `ImageOps.contain` 或者等比例缩放居中裁剪模式。
- **长文本溢出：** 如果有描述文字过长，`wrapped_centered` 排版依然可能超出底部 `DESC_BOT`。可以通过测试调小 `SIZE_DESC` 或增加行宽容错解决。
